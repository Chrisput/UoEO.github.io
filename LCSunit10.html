Unit 10: Data Representation using Database Concepts

On completion of this unit I am able to

  • Apply database concepts and theories in the database design in terms of database integrity and data consistency.
  • Create and manipulate data stored in the database by means of the structured query language, SQL.
  • Review the security and maintenance methods of a database.

Reflections

Below are my key learnings and reflections

  • There are several ways to define a database and dataset, i.e. Flat File, Relational, Hierarchical, Network, and Object-Oriented Model
  • Three Tier architecture, i.e. application layer, database mangement system (DBMS), and database. DBMS is an abstract tool that manipulates the actual database.
  • SQL is a query language for relational database systems MySQL. Query statements: SELECT - extract rows from a relation, PROJECT - extract columns from a relation, JOIN - combine different relations into one relation.
  • Database integrity is maintained through the commit/rollback protocol and through locking protocols.
  • Data Mining is practiced on data warehouses and has different formsL Class description/discrimination, Cluster analysis, Association analysis, Outlier analysis, Sequential patterns analysis.
